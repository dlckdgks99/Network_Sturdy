# 1.1 컴퓨터 통신 프로토콜의 정의 
### 통신 프로토콜의 특징
1. 효율적이고 안정적으로 통신할 수 있도록 미리 정한 약속
  - 효율적인 통신 : 주어진 통신 채널(매체, 전송 속도 등)을 최대로 이용할 수 있도록 프로토콜이 정해져야 함
    - 프로토콜의 처리 순서가 잘 정의, 패킷의 최적크기 설정, 흐름 제어
  - 안정적인 통신 : 비정상적인 외부 요인이 있어도 계속 안정되게 동작하여햐 하며 필요하다면 안전하게 통신이 종료되어야 한다.
    - 에러제어 필요

### 통신 프로토콜의 종류
1. 표준 통신 프로토콜 : TCP/IP, http, telnet, ftp, IEEE 802.3, 이더넷, 무선 LAN(IEEE 802.11)
2. 사용자 정의 통신 프로토콜 : 특정한 서비스를 제공하기 위해 개발자가 임의로 정의한 통신 프로토콜(EX 온라인 수강신청)

### OSI 7 계층 구조
1. 물리 계층
  - 정보의 최소 단위인 비트를 전송매체를 통하여 효율적으로 전송하는 기능을 정의
2. 링크 계층
  - 물리 계층의 서비스를 이용하여 다수의 비트로 구성된 데이터 블록인 프레임(Frame)을 노드 사이에 신뢰성 있게 전송하는 기능을 정의
  - 링크의 개설과 해제, 프레임의 시작과 끝을 찾는 프레임 경계식별, 에러제어, 흐름제어 등을 수행
  - 프레임의 전달을 담당
3. 네트워크 계층
  - 링크 계층에 의하여 프레임에 실려 온 패킷을 라우터 등의 교환 장치들을 거쳐 목적지 호스트까지 전달하는 교환 기능을 제공
  - 주소의 분석, 호스트 사이의 논리적 연결설정과 해제, 패킷 단위의 흐름어, 경로 배정(routing)등을 수행
  - 패킷의 교환 기능만 처리
4. 트랜스포트 계층
  - 계층 3을 이용하여 종점 호스트 사이의 데이터 송수신을 다룸
  - 종점간 연결관리, 종점간 에러제어와 종점간 흐름제어
5. 세션 계층
  - 계층 4를 이용하여 통신 서비스의 개설, 서비스 유지 및 종료를 담당
  - 종점 호스트 프로세스에서 이루어지는 통신 관리 프로토콜
6. 표현 계층
  - 데이터의 표현 방식이 서로 다른 호스트들 사이의 통신을 지원하는 프로토콜
  - 데이터의 표준화된 표현 방식의 사용, 코드변환, 데이터 압축 등을 수행.
  - 데이터의 암호화와 복호화
7. 응용 계층
- 네트워크를 이용한 최종 응용 서비스를 제공하는 계층
- ftp, telnet,e-mail, 웹 서비스, 망관리, 분산처리와 같은 네트워크 응용 기능을 처리
  
### TCP/IP 개요
1. TCP/IP 프로토콜 스택
  - 네트워크 액세스 계층
    - IP 데이터그램을 전달하기 위해 이더넷, DSL 등의 서브네트워크를 이용하는 프로토콜
  - 인터넷 계층
    - IP 계층이라고도 하는데 IP 데이터그램을 최종 목적지까지 전달하는 기능을 담당
    - IP 계층의 핵심 기능은 IP 데이터그램을 어떤 경로로 전달할지를 정하는 라우팅
    - ICMP는 IP 데이터그램의 전송을 돕는 제어 프로토콜로서 호스트 또는 라우터 사이에 에러 정보나 제어 정보를 전달하는 데 사용되며 주로 IP 프로토콜이 이용하지만 ping이나 traceroute 같은 응용 프로그램이 사용
     - IGMP는 멀티 캐스팅을 지원하는 프로토콜
  - 트랜스포트 계층
    - TCP는 신뢰성 있는 연결형 서비스가 필요할 때 사용되며 ftp, telnet, e-mail, http 등에서 tcp사용
    - tcp는 신뢰성 있는 통신을 제공하기 위해서 확인 응답(ACK), checksum, 재전송, 종점간 흐름제어 등을 사용
    - UDP는 비연결형 트랜스포트 서비스를 제공
    - 연결설정이나 연결종료 과정이 없으며 목적지로 바로 IP 데이터그램을 전송
    - 데이터그램 단위로 송수신이 이루어짐, 헤더의 크기가 tcp보다 작고, 연결 지연이 없음 
  -응용 계층
    - 트랜스포트 계층으로 TCP와 UDP 중 어느 것을 사용하는 지에 따라 분류
      
2. 인터넷과 서브네트워크
  - 인터넷을 이용하기 위해서 호스트에는 tcp/ip 프로토콜이 설치되어 있어야 하고, 호스트는 라우터들을 경유하여 서로 연결
  - 호스트가 인터넷에 물리적으로 접속하기 위해서 네트워크 액세스 프로토콜이 필요
  - 서브네트워크는 이더넷,패킷 교환망, DSL, ATM 등 데이터를 실제로 전달해주는 네트워크

    
### 통신 프로토콜 표현 방식
1. 필드정의
  - SDU 앞에 붙는 것을 헤데, SDU 붙는 것을 트레일러
  - 헤더나 트레일러 내의 지정된 필드에 프로토콜 처리 내용을 기록하는 방식이 바로 필드 정의에 의한 프로토콜 표현 방식
2. 파라미터 정의
  - 시간 종료 에러가 발샐ㅇ하도록 한 경우에 이 기다리는 시간 값이 바로 파라미터
3. 상태 정의
  - 과거의 동작 결과를 바탕으로 얻어진 현재의 동작 환경을 말함
4. 동작 내용 정의
  - 어떤 상태에서 어떤 입력을 받았을 때 어떤 작업을 수행한다
  - 상위 계층으로부터의 명령, 하위 계층으로부터의 서비스, 자체 이벤트
    
### 네트워크 프로그래밍 계층별 분류
1. 응용 계층 프로그래밍
  - 네트워크 응용 패키지나 TCP/IP 응용 프로토콜을 활용하는 프로그래밍을 말함
  - rsh, rcp, RPC, http를 이용하는 웹, CORBA
  - 복잡한 네트워크 서비스를 편리하게 구현할 수 있다는 장점
2. 트랜스포트 프로그래밍
  - TCP나 UDP와 같은 트랜스포트 계층의 기능을 직접 이용하는 프로그래밍을 말하며 특히 TCP를 이용할 경우 종점 호스트 사이의 연결성, 흐름제어,에러제어 등을 이용할 수 있음
  - 소켓(Socket) : 대표적인 API
3. 디바이스 드라이버 계층 프로그래밍
  - OSI 계층 2 이하의 인터페이스, 즉 링크 계층이나 하드웨어 디바이스를 통해 프레임 단위의 데이터 송수신을 직접 다루는 프로그래밍
